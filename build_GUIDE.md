# 정적 웹사이트 주석 제거 및 Minify 빌드 완료 보고서

보안 취약점 방지와 리소스 로딩 속도 최적화를 위해, 추가 프로그램 설치 없이 윈도우 기본 PowerShell 환경에서 즉시 실행 가능한 정적 페이지 빌드 및 주석 제거 자동화 스크립트(`build.ps1`) 구축을 완료하였습니다.

---

## 1. 수행 결과 요약

*   **빌드 도구(`build.ps1`):** 
    *   .NET IO API(`[System.IO.Directory]`, `[System.IO.File]`)를 기반으로 설계하여, 윈도우 로컬의 대괄호가 섞인 특수 파일 경로에서도 와일드카드 오류 없이 안전하게 파일 복사 및 입출력이 가능하도록 조치했습니다.
    *   로컬 개발 소스코드는 주석이 포함된 상태로 보존하며, 빌드 실행 시 `dist/` 폴더를 새로 만들고 주석이 삭제/압축된 배포용 산출물을 모아줍니다.
*   **주석 제거 및 압축 검증:**
    *   `dist/index.html` 내의 `<!-- ... -->` 형태의 모든 구조적 주석이 정상적으로 소거되고 빈 줄이 정리되었습니다.
    *   `dist/css/*.css` 파일의 주석이 제거되고 개행과 띄어쓰기가 한 줄로 축소되어 로딩 성능이 최적화되었습니다.
    *   `dist/js/*.js` 파일들의 주석이 코드가 망가지지 않도록 안전한 정규식으로 지워졌으며, 특히 URL 문자열 내의 `//`나 작은따옴표 내의 리터럴은 정상 보존됨을 확인하였습니다.

## 3. 웹 배포 가이드 (사용법)

앞으로 웹서버 또는 GitHub Pages에 사이트를 배포하거나 수정사항을 반영하실 때 아래의 흐름을 따르시면 됩니다.

1.  **개발 작업:**
    *   루트 폴더에 있는 원본 파일(`index.html`, `market.html`, `js/main.js` 등)을 평소처럼 자유롭게 수정하고 주석을 남기며 작업합니다.
2.  **빌드 실행:**
    *   윈도우 PowerShell 창을 열고 아래 명령어를 실행합니다.
    ```powershell
    powershell -ExecutionPolicy Bypass -File .\build.ps1
    ```
    *   실행 완료 시 프로젝트 루트 아래에 `dist/` 폴더가 최신화됩니다.
3.  **배포 (GitHub Pages 동기화):**
    *   `dist/` 폴더가 `.gitignore`에 등록되어 있어 발생하는 배포 충돌을 해결하기 위해, 빌드 스크립트(`build.ps1`) 내부의 마지막 단계에 **임시 깃 구성 및 깃허브 강제 배포 푸시(Force Push) 로직을 통합**해 두었습니다.
    *   **사용 방법:**
        따라서 별도의 Git 배포 명령어를 수동으로 입력하실 필요 없이, 아래의 빌드 실행 명령어를 돌려주기만 하면 **코드 압축/주석 제거와 실서버 배포가 한 번에 완료**됩니다.
        ```powershell
        powershell -ExecutionPolicy Bypass -File .\build.ps1
        ```
    *   **GitHub Pages 최초 활성화 설정 (최초 1회만 수행):**
        1. GitHub 내 해당 저장소(Repository)로 이동하여 상단의 **Settings** 탭을 클릭합니다.
        2. 왼쪽 사이드바 메뉴에서 **Pages**를 선택합니다.
        3. **Build and deployment** 섹션의 Branch 항목에서 **`gh-pages`** 브랜치와 **`/ (root)`** 폴더를 지정한 후 **Save** 버튼을 누릅니다.
